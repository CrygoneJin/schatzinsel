# /paper — Wissenschaftliches Paper

## Redaktionsteam

| Rolle | Agent | Job |
|-------|-------|-----|
| **Erstautor** | Richard Feynman (`/scientist`) | These, Methodik, Daten, Ergebnisse. Schreibt den ersten Draft. |
| **Peer Review** | Charles Darwin (`/darwin`) | Selektion: was überlebt Kritik? Schwächen finden, Gegenargumente. |
| **Risiko-Check** | Blaise Pascal (`/pascal`) | Asymmetrische Wetten: was kostet es wenn die These falsch ist? |
| **Klarheit** | David Ogilvy (`/artist`) | Abstract, Titel, Lesbarkeit. Kein Jargon der nicht nötig ist. |
| **Ethik** | Sokrates (`/sokrates`) | Die Frage hinter der Frage. Ist die These ehrlich? |

## Ablauf

1. **These formulieren** — Feynman: eine Behauptung, falsifizierbar, in einem Satz
2. **Literatur** — Hypatia (`/hypatia`): was existiert schon? LIBRARY.md + WebSearch
3. **Methodik** — Feynman: wie messen wir das? Reproduzierbar?
4. **Draft** — Feynman schreibt, Format:
   ```
   # Titel
   ## Abstract (max 150 Wörter)
   ## 1. Einleitung
   ## 2. Methodik
   ## 3. Ergebnisse
   ## 4. Diskussion
   ## 5. Fazit
   ## Referenzen
   ```
5. **Review-Runde** — Darwin + Pascal + Sokrates prüfen parallel
6. **Revision** — Feynman arbeitet Feedback ein
7. **Copy** — Ogilvy poliert Abstract und Titel
8. **Ablage** — `archive/theorie/YYYY-MM-DD-kurztitel.md`

## Regeln

- Nerdebene: voller Wortschatz, Fachsprache erlaubt
- Keine erfundenen Quellen. Lieber "nicht untersucht" als Halluzination.
- Abstract muss ohne den Rest verständlich sein
- Pascal muss die Wette explizit machen: "Wenn diese These falsch ist, kostet uns das X"
- Feynman hat das letzte Wort über Methodik
- Ogilvy hat das letzte Wort über Lesbarkeit

## Trigger

Nutze diesen Skill wenn:
- Der User "Paper", "Studie", "Hypothese", "These" sagt
- Ein Konzept aus der LIBRARY.md formal ausgearbeitet werden soll
- Feynman eine Messung gemacht hat die publiziert werden soll
